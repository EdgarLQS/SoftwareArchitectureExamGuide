# 面向对象设计

## SOLID 五大设计原则

| 原则 | 英文 | 含义 | 典型应用 | 反面案例 |
|------|------|------|---------|---------|
| **单一职责** | SRP | 一个类只引起一个变化原因 | 用户类只负责用户数据，不负责日志 | God Class（上帝类） |
| **开闭原则** | OCP | 对扩展开放，对修改关闭 | 通过接口扩展支付方式而不改核心代码 | 大量 if-else 判断类型 |
| **里氏替换** | LSP | 子类必须能替换父类 | 正方形不能继承矩形（破坏行为预期） | 子类重写方法改变语义 |
| **接口隔离** | ISP | 客户端不应依赖不需要的接口 | 拆分大接口为小接口 | 胖接口/万能接口 |
| **依赖倒置** | DIP | 高层模块不应依赖低层模块，都依赖抽象 | 依赖注入、控制反转 | 直接 new 具体实现类 |

> **核心记忆**：SRP 解决"做什么"，OCP 解决"怎么扩"，LSP 解决"怎么继承"，ISP 解决"怎么设计接口"，DIP 解决"怎么解耦"。

## 补充设计原则

| 原则 | 含义 | 与 SOLID 的关系 |
|------|------|----------------|
| **DRY** | Don't Repeat Yourself，不重复知识 | 与 SRP 配合，消除重复代码 |
| **KISS** | Keep It Simple Stupid，保持简单 | 指导 OCP 的实现方式 |
| **LOD（迪米特法则）** | 只与直接朋友通信，降低耦合 | 与 ISP 配合减少依赖 |
| **组合优于继承** | 优先使用组合而非继承 | 避免 LSP 被破坏 |

## 23 种设计模式分类

### 创建型（5 种）— 关注对象的创建过程

| 模式 | 一句话定义 | 典型应用场景 | 软考关联 |
|------|-----------|-------------|---------|
| **单例** | 保证一个类只有一个实例 | 配置管理器、连接池、日志管理器 | 数据库连接池 |
| **工厂方法** | 定义创建对象的接口，子类决定创建哪种 | 日志工厂（文件/数据库/远程） | SOA 服务创建 |
| **抽象工厂** | 创建一系列相关或依赖对象 | UI 主题工厂（Windows/Mac/Linux 风格） | 多数据源适配 |
| **建造者** | 分步骤构建复杂对象 | 构建复杂请求对象、SQL 构造器 | 报表生成器 |
| **原型** | 通过复制已有实例创建新对象 | 对象克隆、配置模板复制 | 缓存对象复制 |

### 结构型（7 种）— 关注类与对象的组合

| 模式 | 一句话定义 | 典型应用场景 | 软考关联 |
|------|-----------|-------------|---------|
| **适配器** | 将接口转换为客户期望的另一个接口 | 旧系统接口适配、多源数据格式转换 | 多源异构数据集成 |
| **装饰器** | 动态地给对象添加功能 | 缓存装饰器、日志装饰器、权限校验 | AOP 切面实现 |
| **代理** | 为其他对象提供占位以控制访问 | 远程代理、虚拟代理、保护代理 | API 网关、微服务治理 |
| **外观** | 为子系统提供统一的高层接口 | 统一入口封装多个子系统的复杂调用 | 系统集成 |
| **桥接** | 将抽象与实现分离，使它们可独立变化 | 跨平台 GUI、多渠道消息发送 | 企业集成平台 |
| **组合** | 将对象组合成树形结构表示部分-整体 | 文件系统、组织架构树、菜单组件 | UML 建模 |
| **享元** | 共享细粒度对象以减少内存开销 | 字符串常量池、线程池、连接池 | 高并发性能优化 |

### 行为型（11 种）— 关注对象间的通信与职责分配

| 模式 | 一句话定义 | 典型应用场景 | 软考关联 |
|------|-----------|-------------|---------|
| **策略** | 定义一族算法，使它们可互换 | 负载均衡算法、支付策略、排序策略 | 2025上 负载均衡 |
| **模板方法** | 定义算法骨架，子类实现具体步骤 | 测试框架、报表生成流程 | 2024上 单元测试 |
| **命令** | 将请求封装为对象，支持撤销 | 事务操作、命令队列、宏命令 | 工作流引擎 |
| **状态** | 对象内部状态改变时行为改变 | 订单状态机、审批流程 | 业务流程管理 |
| **责任链** | 请求沿处理链传递，每个节点决定是否处理 | 审批链、拦截器链、过滤器链 | API 网关 |
| **中介者** | 用一个中介对象封装一组对象交互 | 聊天室、GUI 对话框协调 | 微服务编排 |
| **迭代器** | 顺序访问集合元素而不暴露内部结构 | 集合遍历、游标遍历 | 数据库结果集 |
| **访问者** | 在不修改数据结构的前提下定义新操作 | AST 遍历、报表统计 | 代码分析工具 |
| **备忘录** | 捕获并外部化对象内部状态，支持恢复 | 撤销/重做、快照、事务回滚 | 容错恢复 |
| **解释器** | 定义语言文法并解释执行 | 规则引擎、SQL 解析、正则表达式 | 业务规则引擎 |
| **观察者** | 一对多依赖，状态改变时所有依赖者收到通知 | 事件监听、消息订阅、响应式编程 | 2025上 事件驱动 |

## AOP 面向切面编程（2021年论文真题）

### AOP 核心概念

| 概念 | 说明 |
|------|------|
| **横切关注点** | 跨越多个模块的公共功能（日志、事务、权限、缓存） |
| **切面（Aspect）** | 横切关注点的模块化封装 |
| **通知（Advice）** | 切面在特定连接点执行的动作 |
| **连接点（JoinPoint）** | 程序执行过程中的某个点（方法调用、异常抛出） |
| **切入点（Pointcut）** | 匹配连接点的表达式，决定通知在哪里执行 |
| **织入（Weaving）** | 将切面应用到目标对象创建代理对象的过程 |

### 五种通知类型

| 通知类型 | 执行时机 | 用途 |
|---------|---------|------|
| **Before（前置通知）** | 目标方法执行前 | 参数校验、权限检查 |
| **After（后置通知）** | 目标方法执行后（无论是否异常） | 资源释放、日志记录 |
| **Around（环绕通知）** | 包裹目标方法，可控制是否执行 | 性能监控、事务管理 |
| **AfterReturning（返回通知）** | 目标方法正常返回后 | 结果处理、缓存更新 |
| **AfterThrowing（异常通知）** | 目标方法抛出异常后 | 异常处理、告警通知 |

### AOP 织入方式

| 方式 | 时机 | 代表技术 |
|------|------|---------|
| **编译时织入** | 编译阶段 | AspectJ |
| **类加载时织入** | 类加载阶段 | AspectJ LTW |
| **运行时织入** | 运行阶段（动态代理） | Spring AOP（JDK 动态代理 / CGLIB） |

### AOP vs OOP

| 维度 | OOP | AOP |
|------|-----|-----|
| 关注点 | 业务逻辑的纵向分解 | 横切关注点的横向抽取 |
| 封装单位 | 类/对象 | 切面 |
| 解决的问题 | 业务模块化 | 公共逻辑复用 |
| 关系 | AOP 是 OOP 的补充，非替代 | |

### AOP 典型应用

- **日志切面**：记录方法入参、出参、执行时间
- **事务切面**：声明式事务管理（@Transactional）
- **权限切面**：接口访问控制
- **缓存切面**：方法结果缓存
- **性能监控切面**：接口响应时间统计

## GRASP 通用职责分配模式

| 模式 | 职责 |
|------|------|
| 信息专家 | 职责分配给拥有完成该职责所需信息的类 |
| 创建者 | 创建职责分配给包含或聚合被创建对象的类 |
| 低耦合 | 职责分配使类之间依赖最小化 |
| 高内聚 | 职责分配使类的职责紧密相关 |
| 控制器 | 将处理系统事件的职责分配给非 UI 类 |
| 多态 | 根据类型多态分配职责 |
| 纯虚构 | 创建不存在但有助于降低耦合的虚拟类 |
| 间接性 | 通过中间类解耦 |
| 防止变异 | 将易变职责封装到稳定接口中 |
