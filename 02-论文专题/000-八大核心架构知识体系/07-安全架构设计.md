# 安全架构设计

## 安全模型

### 经典安全模型对比

| 模型 | 保护目标 | 核心规则 | 适用场景 |
|------|---------|---------|---------|
| **BLP（Bell-LaPadula）** | 保密性（Confidentiality） | No Read Up（不能读更高密级）<br>No Write Down（不能写更低密级） | 军事系统、政府机密 |
| **Biba** | 完整性（Integrity） | No Read Down（不能读更低完整级别）<br>No Write Up（不能写更高完整级别） | 金融系统、数据完整性要求高的系统 |
| **Chinese Wall** | 利益冲突隔离 | 同一顾问不能访问同一利益冲突组中的两个不同客户数据 | 金融咨询、审计系统 |

### BLP 模型详解

| 规则 | 说明 | 目的 |
|------|------|------|
| **简单安全规则** | 主体不能读取密级高于自身的数据 | 防止信息泄露 |
| **星号规则（\*-property）** | 主体不能写入密级低于自身的数据 | 防止高密级信息流向低密级 |
| **自主安全规则** | 基于访问矩阵的自主访问控制 | 灵活授权 |

### Biba 模型详解

| 规则 | 说明 | 目的 |
|------|------|------|
| **简单完整性规则** | 主体不能读取完整级别低于自身的数据 | 防止低完整数据污染 |
| **完整性星号规则** | 主体不能写入完整级别高于自身的数据 | 防止污染高完整数据 |
| **调用完整性规则** | 主体不能调用完整级别低于自身的程序 | 防止低完整程序影响 |

## 访问控制模型

### 四种访问控制模型对比

| 模型 | 全称 | 控制主体 | 特点 | 适用场景 |
|------|------|---------|------|---------|
| **DAC** | 自主访问控制 | 资源所有者 | 资源所有者决定谁可以访问 | 个人文件系统 |
| **MAC** | 强制访问控制 | 系统/管理员 | 基于安全标签强制控制 | 军事/政府系统 |
| **RBAC** | 基于角色的访问控制 | 角色管理员 | 权限分配给角色，用户分配角色 | 企业信息系统 |
| **ABAC** | 基于属性的访问控制 | 策略引擎 | 基于用户/资源/环境属性动态决策 | 云环境、零信任 |

### RBAC 模型要素

| 要素 | 说明 |
|------|------|
| **用户（User）** | 系统的使用者 |
| **角色（Role）** | 权限的集合 |
| **权限（Permission）** | 对资源的操作许可 |
| **会话（Session）** | 用户激活的角色子集 |

### RBAC 层级

| 层级 | 说明 |
|------|------|
| **RBAC0** | 基础 RBAC（用户-角色-权限） |
| **RBAC1** | RBAC0 + 角色继承 |
| **RBAC2** | RBAC0 + 约束（互斥角色、基数约束） |
| **RBAC3** | RBAC1 + RBAC2（角色继承 + 约束） |

### ABAC 基于属性的访问控制

| 属性类型 | 示例 |
|---------|------|
| **主体属性** | 用户角色、部门、安全等级 |
| **资源属性** | 文件类型、密级、所有者 |
| **环境属性** | 时间、地点、网络位置 |
| **操作属性** | 读、写、删除、执行 |

**优势**：比 RBAC 更灵活，支持细粒度和动态访问控制。

## 安全体系架构 WPDRRC

### 六阶段详解

| 阶段 | 说明 | 技术手段 |
|------|------|---------|
| **预警（Warning）** | 预测和分析潜在安全威胁 | 威胁情报、漏洞扫描、风险评估 |
| **保护（Protection）** | 防止安全事件发生 | 防火墙、加密、访问控制、身份认证 |
| **检测（Detection）** | 发现正在进行的安全威胁 | IDS/IPS、日志分析、异常检测 |
| **响应（Response）** | 安全事件发生后的应对措施 | 事件响应流程、告警、隔离 |
| **恢复（Recovery）** | 从安全事件中恢复正常 | 数据备份、灾难恢复、业务恢复 |
| **反击（Counterattack）** | 溯源并反制攻击者 | 威胁溯源、蜜罐、威胁情报共享 |

### WPDRRC vs P2DR

| 模型 | 阶段 | 特点 |
|------|------|------|
| **WPDRRC** | 预警→保护→检测→响应→恢复→反击 | 更全面，含预警和反击 |
| **P2DR** | 策略→保护→检测→响应 | 以安全策略为核心 |

## 加密技术

### 对称加密

| 算法 | 密钥长度 | 特点 | 适用场景 |
|------|---------|------|---------|
| **DES** | 56 位 | 已不安全，仅用于兼容 | 遗留系统 |
| **3DES** | 168 位 | DES 的三重加密 | 过渡方案 |
| **AES** | 128/192/256 位 | 快速安全，NIST 标准 | 主流选择 |

### 非对称加密

| 算法 | 特点 | 适用场景 |
|------|------|---------|
| **RSA** | 大整数分解困难，密钥较长（2048+ 位） | 数字签名、密钥交换 |
| **ECC** | 椭圆曲线，同等安全密钥更短 | 移动设备、IoT |
| **DSA** | 仅用于签名，不加密 | 数字签名 |

### 哈希算法

| 算法 | 摘要长度 | 安全性 | 说明 |
|------|---------|--------|------|
| **MD5** | 128 位 | 已不安全（碰撞攻击） | 仅用于校验 |
| **SHA-1** | 160 位 | 已不安全 | 逐步淘汰 |
| **SHA-256** | 256 位 | 安全 | 主流选择 |
| **SHA-512** | 512 位 | 更安全 | 高安全需求 |

### 数字签名与数字证书

- **数字签名**：发送方用私钥对消息摘要签名，接收方用公钥验证 → 保证完整性和不可抵赖性
- **数字证书**：由 CA 签发的公钥证书（X.509 格式），绑定公钥与持有者身份
- **PKI（公钥基础设施）**：CA、RA、证书库、CRL 组成的信任基础设施

## 身份鉴别（2021年论文真题关联）

### 认证方式

| 方式 | 说明 | 安全等级 |
|------|------|---------|
| **单因素认证** | 仅密码 | 低 |
| **多因素认证（MFA）** | 密码 + 短信/Token/生物 | 高 |

### 单点登录 SSO

用户一次登录后，可在多个关联系统中自由访问而无需重新认证。

| 方案 | 说明 |
|------|------|
| **基于 Cookie** | 同一域名下共享 Cookie |
| **基于 Token** | CAS、OAuth 2.0 |
| **基于 SAML** | 企业级 SSO，XML 格式 |

### OAuth 2.0 授权框架

| 角色 | 说明 |
|------|------|
| **资源所有者** | 用户 |
| **客户端** | 第三方应用 |
| **授权服务器** | 颁发访问令牌 |
| **资源服务器** | 存储受保护资源 |

| 授权模式 | 适用场景 |
|---------|---------|
| **授权码模式** | 服务端 Web 应用（最安全） |
| **隐式模式** | 纯前端应用（已不推荐） |
| **客户端凭证模式** | 服务间通信 |
| **密码模式** | 信任应用（已不推荐） |

### OpenID Connect（OIDC）

基于 OAuth 2.0 的身份认证协议，在 OAuth 基础上增加 ID Token（JWT 格式），提供用户身份信息。

### JWT（JSON Web Token）

| 部分 | 说明 |
|------|------|
| **Header** | 算法和令牌类型 |
| **Payload** | 用户声明信息（claims） |
| **Signature** | 用密钥签名，防止篡改 |

**优点**：无状态、跨域、自包含。
**注意**：Payload 仅 Base64 编码而非加密，不可存敏感信息。

## 零信任架构

### 核心原则

- **永不信任，始终验证**：不区分内网/外网，所有访问都需认证和授权
- **最小权限**：仅授予完成工作所需的最小权限
- **持续验证**：不是一次认证，而是持续评估信任度

### 零信任 vs 传统边界安全

| 维度 | 传统边界安全 | 零信任 |
|------|-------------|--------|
| 信任模型 | 内网可信、外网不可信 | 全都不信任 |
| 防护重点 | 网络边界 | 身份和数据 |
| 访问控制 | 粗粒度（网络层） | 细粒度（应用/数据层） |
| 安全假设 | 内网安全 | 内网也可能被入侵 |

### 零信任关键技术

| 技术 | 说明 |
|------|------|
| **微隔离** | 将网络细分为小区域，独立控制 |
| **持续验证** | 基于行为、设备状态、环境持续评估 |
| **最小权限访问** | 动态授权，用完即撤销 |
| **端到端加密** | 所有通信加密 |

## 安全架构设计原则

| 原则 | 说明 |
|------|------|
| **纵深防御** | 多层防护，一层失效其他层仍有效 |
| **最小权限** | 只授予完成工作所需的最小权限 |
| **默认拒绝** | 未明确允许的即为禁止 |
| **安全开发生命周期（SDL）** | 在开发各阶段嵌入安全实践 |

## Web 安全

### OWASP Top 10（2021）

| 排名 | 漏洞 | 防御措施 |
|------|------|---------|
| 1 | **访问控制失效** | RBAC/ABAC、最小权限 |
| 2 | **加密机制失效** | TLS、敏感数据加密 |
| 3 | **注入** | 参数化查询、输入验证 |
| 4 | **不安全的设计** | 威胁建模、安全设计模式 |
| 5 | **安全配置错误** | 默认安全配置、自动化检查 |
| 6 | **自带漏洞的组件** | 组件更新、漏洞扫描 |
| 7 | **识别与认证失败** | MFA、密码策略、会话管理 |
| 8 | **软件与数据完整性漏洞** | 代码签名、CI/CD 安全 |
| 9 | **安全日志与监控不足** | 日志记录、告警响应 |
| 10 | **服务端请求伪造（SSRF）** | URL 白名单、网络隔离 |

### SQL 注入防御

- 参数化查询（预编译语句）
- 输入验证（白名单）
- 最小权限数据库账户

### XSS（跨站脚本）防御

- 输出编码/转义
- 内容安全策略（CSP）
- HttpOnly Cookie 防止 JS 读取

### CSRF（跨站请求伪造）防御

- 使用 CSRF Token
- SameSite Cookie 属性
- 验证 Referer Header
