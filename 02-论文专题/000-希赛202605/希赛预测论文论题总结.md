### 论题 1：论软件架构风格

**题目三问：**
1. 概要叙述你参与分析和设计的软件系统开发项目以及你所担任的主要工作
2. 软件系统开发中常用的软件架构风格有哪些？详细阐述每种风格的具体含义
3. 详细说明你所参与分析和设计的软件系统是采用什么软件架构风格的，并分析采用该架构风格设计的原因

**核心知识速查：**

Garlan & Shaw 五大分类：

| 大类 | 子风格 | 关键词 |
|------|--------|--------|
| 数据流风格 | 批处理序列、管道/过滤器 | 数据传递、增量处理 |
| 调用/返回风格 | 主程序/子程序、面向对象、层次结构 | 过程调用、封装、分层 |
| 独立构件风格 | 进程通信、事件驱动 | 消息传递、隐式调用 |
| 虚拟机风格 | 解释器、基于规则的系统 | 模拟执行、规则匹配 |
| 仓库风格 | 数据库系统、黑板系统 | 数据中心、知识源协同 |

其他：C/S、B/S、CORBA、DCOM、EJB、状态转移、特定领域架构

**各子风格详细含义：**

- **批处理序列**：组件为一系列固定顺序的计算单元，组件间只通过数据传递交互。每个处理步骤是独立的程序，每一步必须在前一步结束后才能开始，数据必须是完整的，以整体的方式传递
- **管道/过滤器**：每个构件都有一组输入和输出，构件读输入数据流，经过内部处理，然后产生输出数据流。这个过程通常通过对输入流的变换及增量计算来完成。在输入被完全消费之前，输出便产生了。构件被称为过滤器，连接件就是数据流传输的管道
- **主程序/子程序**：单线程控制，把问题划分为若干处理步骤，构件即为主程序和子程序。过程调用作为交互机制，即充当连接件。调用关系具有层次性，其语义逻辑表现为主程序的正确性取决于它调用的子程序的正确性
- **面向对象**：构件是对象，对象是抽象数据类型的实例。在抽象数据类型中，数据的表示和它们的相应操作被封装起来。对象是通过函数和过程的调用来交互的。对象具有封装性，一个对象的改变不会影响其他对象
- **层次结构**：层次系统组织成一个层次结构。每层为上一层提供服务，使用下一层的服务，只能见到与自己邻接的层。大的问题分解为若干个渐进的小问题，逐步解决，隐藏了很多复杂度。修改一层，最多影响两层，而通常只能影响上层
- **进程通信**：构件是独立的过程，连接件是消息传递。构件通常是命名过程，消息传递的方式可以是点对点、异步和同步方式、以及远过程调用等
- **事件驱动**：构件不直接调用一个过程，而是触发或广播一个或多个事件。系统中的其他构件中的过程在一个或多个事件中注册，当一个事件被触发，系统自动调用在这个事件中注册的所有过程。基于隐式调用（Implicit Invocation）实现。主要优点是为软件重用提供了强大的支持，为构件的维护和演化带来了方便
- **解释器**：虚拟机风格，模拟解释执行
- **数据库系统**：仓库风格，以数据库为中心
- **黑板系统**：仓库风格，多知识源协同求解

**写作策略：**
- 第二部分选 2-3 个风格类别即可，不是文章重心
- 第三部分是核心：说明为什么选该风格、该风格如何解决项目中的具体问题
- 避坑：不要简单罗列所有风格，重点论述项目实际使用的风格

**对应范文：** `0001-软件架构风格/03-软件架构风格论文范文.md`

### 论题 3：论系统自动化测试及其应用

**题目三问：**
1. 概要叙述你参与管理和开发的软件项目以及你在其中所担任的主要工作
2. 详细论述系统自动化测试的主要工作内容及优缺点
3. 结合你具体参与管理和开发的实际项目，说明是如何进行系统自动化测试的，具体实施过程以及应用效果

**核心知识速查：**

**系统测试定义：** 系统测试的对象是完整的、集成的计算机系统。系统测试的目的是验证完整的软件配置项能否和系统正确连接，并满足系统设计文档和软件开发合同规定的要求。系统测试工作任务难度高、工作量大，存在大量重复性工作。

**基础设施：** 文件服务器 + 数据库服务器 + 测试运行环境 + 控制服务器 + Web 服务器 + 客户端程序

**五种实现方法：**

| 方法 | 说明 |
|------|------|
| 代码静态与动态分析 | 分析代码质量和运行时行为 |
| 测试过程的捕获与回放 | 录制用户操作，回放执行 |
| 测试脚本技术 | 编写自动化测试脚本 |
| 虚拟用户技术 | 模拟多用户并发操作 |
| 测试管理技术 | 用例管理、执行调度、结果分析 |

**五大优点：** 提高执行速度（5s vs 50s）、提高工作效率、保证结果准确性、24小时连续运行、模拟并发场景

**三大局限：** 不能取代手工测试、发现的缺陷不如手工、工具无想象力

**写作策略：**
- 第二部分知识性内容重要，需涵盖基础设施+方法+优缺点
- 第三部分写具体实施过程，可用"工具选型 → 脚本编写 → 执行监控 → 效果评价"流程
- 需对自动化测试效果进行量化评价

**对应范文：** ⚠️ 暂无自动化测试范文。`0003-测试/04-系统自动化测试论文写作框架.md` 已提供写作框架，可参照框架自行撰写；另有 `0003-测试/03-系统性能测试论文范文.md` 可参考写作结构

### 论题 5：论基于架构的软件设计方法及应用（ABSD）

**题目三问：**
1. 概要叙述你参与开发的、采用 ABSD 方法的软件项目以及你在其中所承担的主要工作
2. 结合项目实际，详细说明采用 ABSD 方法进行软件开发时，需要经历哪些开发阶段？每个阶段包括哪些主要活动？
3. 阐述你在软件开发的过程中都遇到了哪些实际问题及解决方法

**核心知识速查：ABSD 方法定义**

ABSD 是一个**自顶向下、递归细化**的软件开发方法，以软件系统功能的分解为基础，通过选择架构风格实现质量和商业需求，并强调在架构设计过程中使用**软件架构模板**。采用 ABSD 方法，设计活动可以从项目总体功能框架明确后就开始。

**ABSD 六阶段：**

| 阶段 | 主要活动 | 要点 |
|------|---------|------|
| 1. 架构需求 | 需求获取、标识构件、架构评审 | 功能+质量属性需求 |
| 2. 架构设计 | 提出模型、构件映射、分析交互、产生架构、设计评审 | 迭代过程 |
| 3. 架构文档化 | 生成规格说明书、质量设计说明书 | 整理归档 |
| 4. 架构复审 | 外部人员参加（客户+专家） | 标识风险、发现缺陷 |
| 5. 架构实现 | 架构分析与设计、构件实现、组装、系统测试 | 实现阶段 |
| 6. 架构演化 | 演化计划、构件变动、更新交互、组装测试、技术评审 | 处理需求变更 |

**适用场景：** 不能预先决定所有需求的系统（软件产品线、长生命周期系统）

**常见问题：** 需求优先级筛选、评审意见冲突、技术选型、需求变更处理

**写作策略：**
- 第二部分选择三个阶段详细论述即可，不要面面俱到
- 第三部分必须有"问题 → 解决方案 → 效果"的完整链路
- 要体现架构师独立思考能力，适当总结不足

**对应范文：** `0006-ABSD 基于软件架构设计/03-ABSD 论文范文.md`

### 论题 6：论微服务架构及其应用

**题目三问：**
1. 概要叙述你所参与管理或开发的软件项目，以及你在其中所承担的主要工作
2. 简要描述微服务优点
3. 具体阐述是如何基于微服务架构进行软件设计实现的

**核心知识速查：微服务七大优势**

| 优势 | 一句话说明 |
|------|-----------|
| 技术异构性 | 每个服务可选适合自身的技术栈；新技术可先在单个服务试用再推广 |
| 弹性 | 单体一损俱损，微服务每个服务可内置可用性和降级方案，故障隔离 |
| 扩展 | 单体整体扩展，微服务可针对单个服务独立扩展 |
| 简化部署 | 单体内修改一行代码也需重新部署全部，微服务每个服务独立部署 |
| 与组织匹配 | 架构与组织结构匹配，避免过大代码库，服务所有权可在团队间迁移 |
| 可组合性 | 开放接口供外部使用，可用不同方式构建应用 |
| 可替代性 | 可轻易重写或删除不再使用的服务，单体系统关联性强难以删除 |

**写作策略：**
- 第二部分简要描述 5-7 个优势即可
- 第三部分是重点：服务拆分原则、技术选型、通信机制、服务治理、部署方案
- 现有范文内容全面，可直接参考

**对应范文：** `0004-微服务/03-微服务论文范文.md`

### 论题 7：论系统安全架构设计及其应用

**题目三问：**
1. 概要叙述你参与管理和开发的涉及安全架构设计的软件项目以及承担的主要工作
2. 请详细论述安全架构设计中鉴别框架和访问控制框架设计的内容，并论述鉴别和访问控制所面临的主要威胁有哪些，说明其危害
3. 请简要说明在你所参与项目的开发过程中，在鉴别框架和访问控制框架设计中存在的实际问题，以及是如何解决这些问题的

**核心知识速查：**

**安全保障基础：** 信息系统的安全保障是以**风险和策略**为基础，在信息系统的整个生命周期中提供包括**技术、管理、人员和工程过程**的整体安全。

**信息安全五特征：** 机密性、完整性、可用性、可控性、不可抵赖性

**五种鉴别方式：**

| 方式 | 示例 |
|------|------|
| 已知的 | 秘密口令 |
| 拥有的 | IC 卡、令牌 |
| 不改变的特性 | 生物特征 |
| 第三方建立的 | 递推鉴别 |
| 环境 | 主机地址等 |

**访问控制定义：** 决定开放系统中允许使用哪些资源、在什么地方阻止未授权访问

**写作策略：**
- 第二部分是核心：鉴别框架 + 访问控制框架要分别论述
- 需补充教材知识：威胁类型（口令破解、重放攻击、越权访问等）及危害
- 希赛解析中威胁部分未详细展开，备考时需自行补充

**对应范文：** `0007-系统安全架构设计/03-系统安全架构设计论文范文.md`
