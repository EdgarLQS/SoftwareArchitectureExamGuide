## 一、六大专题知识体系

### 专题一：系统架构设计（高频必考）

#### 1.1 架构风格分类

| 架构风格 | 特点 | 适用场景 |
|---------|------|---------|
| **分层架构** | 层次分明、职责分离 | 企业应用系统 |
| **C/S 架构** | 响应快、交互强 | 桌面应用、实时系统 |
| **B/S 架构** | 零客户端、易维护 | Web 应用、互联网应用 |
| **微服务架构** | 独立部署、服务自治 | 大型复杂系统 |
| **SOA 架构** | 服务复用、ESB 总线 | 企业集成 |
| **管道-过滤器** | 数据流处理 | 编译器、数据处理 |
| **事件驱动架构** | 松耦合、异步处理 | 消息系统、实时通知 |
| **微内核架构** | 核心 + 插件 | IDE、可扩展系统 |

#### 1.2 质量属性（重点掌握）

**六大质量属性定义与实现策略：**

1. **性能（Performance）**：系统响应时间和吞吐量 → 缓存、异步处理、负载均衡、数据分片
2. **可靠性（Reliability）**：系统无故障运行的能力 → 冗余设计、容错机制、错误检测、检查点/回滚
3. **可用性（Availability）**：系统正常提供服务的时间比例 → 集群、故障转移、心跳机制
4. **安全性（Security）**：保护系统免受恶意攻击 → 身份认证、访问控制、加密、审计
5. **可修改性（Modifiability）**：系统易于修改和维护 → 分层、模块化、接口抽象、设计模式
6. **可扩展性（Scalability）**：系统处理增长的能力 → 水平扩展、垂直扩展、微服务

#### 1.3 架构评估方法

| 方法 | 全称 | 用途 | 特点 |
|------|------|------|------|
| **SAAM** | 软件架构分析方法 | 评估可修改性 | 基于场景、定性分析 |
| **ATAM** | 架构权衡分析方法 | 多质量属性评估 | 风险点、敏感点、权衡点 |
| **ALMA** | 架构级别修改分析方法 | 评估修改影响 | SAAM 的扩展 |

**ATAM 评估步骤（记忆）：**
1. 介绍 ATAM 方法 → 2. 介绍业务驱动因素 → 3. 介绍系统架构 → 4. 确定架构方法 → 5. 生成质量属性效用树 → 6. 分析架构方法 → 7. 头脑风暴和优先级排序 → 8. 分析架构方法（再次）→ 9. 呈现结果

#### 2.1 开发方法

|---------|------|---------|

### 专题三：数据库与大数据（高频）

#### 3.1 数据库设计

- **E-R 模型三要素**：实体（矩形）、属性（椭圆）、关系（菱形）
- **关系规范化**：
  - 1NF：属性不可再分
  - 2NF：消除非主属性对码的部分依赖
  - 3NF：消除非主属性对码的传递依赖
  - BCNF：消除主属性对码的部分和传递依赖

#### 3.2 NoSQL 数据库

| 类型 | 代表产品 | 适用场景 |
|------|---------|---------|
| **键值存储** | Redis、Memcached | 缓存、会话管理 |
| **文档存储** | MongoDB、CouchDB | 内容管理、日志 |
| **列族存储** | HBase、Cassandra | 大数据存储 |
| **图数据库** | Neo4j、JanusGraph | 社交网络、知识图谱 |

**Redis 常见考点**：
- 数据类型：String、List、Set、ZSet、Hash
- 持久化：RDB 快照、AOF 日志
- 应用场景：缓存、分布式锁、计数器、排行榜
- 主从复制：全量同步（BGSAVE）、增量同步

#### 3.3 大数据架构

**Lambda 架构**：数据输入 → 批处理层（Batch）+ 速度层（Speed）→ 服务层 → 输出
**Kappa 架构**：数据输入 → 流处理层（Kafka + Flink）→ 输出（纯流处理，简化维护）

### 专题四：Web 应用与云原生（高频）

#### 4.1 Web 架构演进

单体架构 → 垂直拆分 → SOA 架构 → 微服务架构

#### 4.2 微服务核心特征

服务组件化、按业务组织、智能端点哑管道、去中心化治理、去中心化数据管理、基础设施自动化、容错设计

#### 4.3 云原生四大核心技术

1. **容器化**：Docker、Podman
2. **容器编排**：Kubernetes（K8s）
3. **微服务**：Spring Cloud、Service Mesh
4. **DevOps/CI/CD**：Jenkins、GitLab CI、ArgoCD

### 专题五：嵌入式系统（中频）

- **特点**：专用性、实时性、资源受限、可靠性要求高、功耗敏感
- **实时调度**：RMS（速率单调，静态优先级）vs EDF（最早截止期限，动态优先级）
- **软件架构**：前后台系统（查询+中断）、轮询系统、RTOS

### 专题六：信息系统集成（高频）

#### 6.1 企业应用集成（EAI）四个层次

1. **数据级集成**：数据同步、ETL
2. **接口级集成**：API、Web Service
3. **业务流程级集成**：BPM、工作流
4. **界面级集成**：门户、单点登录

#### 6.2 SOA 架构

- **三角色**：服务提供者、服务消费者、服务注册中心
- **关键技术**：Web Service（SOAP、WSDL、UDDI）、ESB、BPEL

#### 6.3 中台架构

- **业务中台**：用户中心、订单中心、支付中心
- **数据中台**：数据仓库、数据湖、数据分析
- **技术中台**：微服务框架、DevOps 平台
